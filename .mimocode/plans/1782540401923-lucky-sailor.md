# Phase 3.3 — Intent Detection

## Goal
AI identifies what the user wants. Hybrid keyword + LLM intent classification, with handlers for all 9 intents.

---

## Current State
- `detect_intent()` in `chat.py` uses 6 simple keyword branches
- Keyword "resume" collides between upload and generate
- 3 intents have no handlers: Generate Resume, Generate Cover Letter, Generate Recruiter Message
- Placement Analytics doesn't exist
- No LLM-based classification fallback

---

## Step 1: Refactor Intent Detection (Hybrid)

**File**: `backend/routers/chat.py` — rewrite `detect_intent()`

**Approach**: Two-phase detection:

1. **Phase 1 — Keyword matching** (fast, free): Check high-confidence keywords first. Each intent gets a priority score. If a single intent matches with high confidence, return it immediately.

2. **Phase 2 — LLM classification** (fallback): If 0 or 2+ intents match, or confidence is low, send the message to the LLM with a classification prompt. The prompt lists all 9 intents with descriptions and asks for a single intent label + extracted parameters.

**Keyword priority order** (to resolve collisions):
1. `upload_resume` — "upload resume", "parse resume", "upload pdf" (explicit upload action)
2. `generate_resume` — "generate resume", "write resume", "create resume", "build resume"
3. `generate_cover_letter` — "cover letter", "write cover letter", "draft cover letter"
4. `generate_recruiter_msg` — "recruiter message", "linkedin message", "outreach message", "cold email"
5. `prepare_interview` — "interview prep", "prepare interview", "practice interview"
6. `analyze_job` — "analyze job", "job description", "analyze this role"
7. `show_applications` — "show applications", "my applications", "application tracker"
8. `show_profile` — "my profile", "my skills", "my experience", "career profile"
9. `placement_analytics` — "analytics", "placement stats", "how am i doing", "progress report"
10. `general_chat` — fallback

**LLM classification prompt** (when needed):
```
Classify this user message into exactly ONE intent.
Intents: upload_resume, generate_resume, generate_cover_letter, generate_recruiter_msg,
         analyze_job, prepare_interview, show_applications, show_profile,
         placement_analytics, general_chat
Return JSON: {"intent": "<intent_name>", "confidence": 0.0-1.0, "params": {}}
User message: "{message}"
```

**New helper**: `classify_intent_with_llm(message)` — calls `generate()` with the classification prompt, parses JSON response. Only called when keyword matching is ambiguous.

## Step 2: Generate Resume Service

**New file**: `backend/services/resume_generator.py`

- System prompt: generates a professional resume in structured Markdown
- Takes: profile data (skills, experience, education, projects), optional target JD
- Returns: formatted resume text
- Output format: **PDF** using `fpdf2` library
- Flow: LLM generates Markdown resume → convert to PDF → return PDF bytes

**New dependency**: `fpdf2` in requirements.txt

**PDF generation helper**:
- Parse Markdown sections (## headers → sections)
- Render with fpdf2: headers in bold, bullet points, clean layout
- Return as `StreamingResponse` with `application/pdf` content type

## Step 3: Generate Cover Letter Service

**New file**: `backend/services/cover_letter.py`

- System prompt: professional cover letter writer
- Takes: profile data, company, role, JD, optional tone (formal/casual)

## Step 4: Generate Recruiter Message Service

**New file**: `backend/services/recruiter_msg.py`

- System prompt: concise outreach message writer (LinkedIn DM / email / cold outreach)
- Takes: profile data, company, role, optional channel (linkedin/email)
- Returns: short message under 150 words
- Handler: overwrites `Application.recruiter_msg`

## Step 5: Placement Analytics Service

**New file**: `backend/services/analytics.py`

**SQL aggregations** (no LLM needed):
- Total applications count
- Status breakdown (applied, interview, rejected, offer, etc.)
- Average match score
- Score distribution (0-20%, 20-40%, 40-60%, 60-80%, 80-100%)
- Top 5 companies targeted
- Most common role titles
- Application timeline (apps per week/month)
- Best/worst match scores

**LLM narrative layer**:
- Takes the SQL results as input
- Generates a natural-language summary with insights and suggestions
- Example: "You've applied to 12 positions with an average match of 72%. Your strongest matches are in frontend roles. Consider targeting more backend positions to diversify..."

## Step 6: Wire Handlers in Chat

**File**: `backend/routers/chat.py` — add 4 new handler branches in `_handle_message()`

Each handler follows the same pattern:
1. Validate prerequisites (profile exists, application exists, etc.)
2. Call the service function
3. Send response via WebSocket (text or stream)
4. Send action event if applicable

**New intents in `_handle_message()`**:
- `generate_resume` → calls `resume_generator.generate_resume()`, sends PDF or text
- `generate_cover_letter` → calls `cover_letter.generate_cover_letter()`, sends text
- `generate_recruiter_msg` → calls `recruiter_msg.generate_recruiter_msg()`, sends text
- `placement_analytics` → calls `analytics.get_analytics()`, sends summary + action
