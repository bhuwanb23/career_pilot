# Phase 3.1 — AI Backend Foundation

## Goal
Make the AI backend configurable, add Gemini as an alternative LLM provider, add proper logging and error handling.

---

## Step 1: Environment Configuration

**Files**: `backend/config.py`, `backend/.env.example`, `backend/requirements.txt`

- Replace hardcoded config with `pydantic-settings` `BaseSettings`
- Add `.env.example` with all vars documented
- Add to `requirements.txt`: `pydantic-settings`, `tenacity`, `google-generativeai`

**Env vars** (with defaults preserving current behavior):
| Var | Default | Purpose |
|-----|---------|---------|
| `LLM_PROVIDER` | `ollama` | `ollama` or `gemini` |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama endpoint |
| `OLLAMA_MODEL` | `llama3.2:1b` | Ollama model name |
| `GEMINI_API_KEY` | (empty) | Google AI API key |
| `GEMINI_MODEL` | `gemini-2.0-flash` | Gemini model name |

| `LOG_LEVEL` | `INFO` | Logging level |
| `CORS_ORIGINS` | `http://localhost:5173,http://localhost:3000` | Allowed origins |

## Step 2: LLM Provider Abstraction

**File**: `backend/services/llm_client.py` — refactor

- Define `LLMProvider` protocol with `generate()`, `generate_stream()`, `health_check()`
- `OllamaProvider` — wraps existing httpx logic, uses **persistent `httpx.AsyncClient`** (connection pooling)
- `GeminiProvider` — uses `google.generativeai`, maps system param to `system_instruction`
- `get_llm_provider()` — factory, reads `settings.LLM_PROVIDER`, singleton
- Public `generate()`, `generate_stream()`, `health_check()` — thin wrappers (preserves import interface)
- Add `tenacity` retry: 3 retries, exponential backoff, on timeout/5xx/server errors

---

## Step 3: Shared JSON Parsing Utility

**New file**: `backend/services/llm_utils.py`

- `strip_markdown_fences(text)` — removes ```json wrappers
- `parse_llm_json(response, fallback)` — strips fences + parses JSON, returns fallback on failure

**Files**: `services/job_analyzer.py`, `services/interview_prep.py`, `services/resume_parser.py`
- Replace duplicated 6-line markdown stripping blocks with `parse_llm_json()`

## Step 4: Logging Setup

**New file**: `backend/logging_config.py`

- `setup_logging()` — configures root logger from `settings.LOG_LEVEL`
- Format: `"%(asctime)s [%(levelname)s] %(name)s: %(message)s"`
- Suppress noisy uvicorn/httpx loggers

**Files to update**:
- `main.py` — call `setup_logging()`, replace `print()` with `logger.info/warning`
- `routers/resume.py`, `routers/applications.py`, `routers/interview.py` — replace `traceback.print_exc()` with `logger.exception()`
- `services/llm_client.py` — log LLM calls (model, prompt length, response length)

---

## Step 5: Error Handling

**New file**: `backend/services/exceptions.py`

- `LLMProviderError` (base)
- `LLMTimeoutError`, `LLMRateLimitError`, `LLMConfigError`

**File**: `routers/chat.py`
- Add `try/except` around entire `while True` loop body
- Catch `LLMProviderError`, `json.JSONDecodeError`, `SQLAlchemyError`
- On error: log it, send `{"type": "error", "content": "..."}` to client, continue loop
- DB rollback on SQLAlchemy errors
