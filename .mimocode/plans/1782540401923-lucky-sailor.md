# Phase 3.4 — Workflow Planner

## Goal
Convert intent into structured execution plans. Instead of monolithic handlers, each intent becomes a sequence of discrete, trackable steps.

---

## Current State
- `_handle_message()` in `chat.py` is a 350-line function with 10 if/elif branches
- Each handler mixes prerequisite checks, LLM calls, DB operations, and response formatting
- No progress tracking — user sees "Analyzing..." then waits for the full result
- No step-level error handling — if an LLM call fails, the whole handler crashes
- No observability — can't tell which step failed or how long each took

---

## Design: Step-Based Workflow Engine

### Core Concepts

**Step**: A discrete unit of work with a name, optional parameters, and an execute function.

**Workflow**: An ordered list of steps for a given intent. Defined as a list of step specs.

**Executor**: Runs a workflow step-by-step, sends progress updates to the client, handles errors per-step.

### Step Types

| Type | Purpose | Example |
|------|---------|---------|
| `check` | Validate prerequisite, abort if missing | "Profile must exist" |
| `prepare` | Transform data for next steps | "Serialize profile dict" |
| `llm_call` | Call an LLM service | "Analyze job description" |
| `db_write` | Persist results | "Save application" |
| `respond` | Format and send response | "Send match score summary" |

### File Structure

```
backend/
  services/
    workflow.py          # Step, Workflow, WorkflowExecutor classes
    workflows/
      __init__.py        # Registry of all workflows
      resume_upload.py   # upload_resume workflow
      resume_generate.py # generate_resume workflow
      cover_letter.py    # generate_cover_letter workflow
      recruiter_msg.py   # generate_recruiter_msg workflow
      analyze_job.py     # analyze_job workflow
      interview_prep.py  # prepare_interview workflow
      show_applications.py
      show_profile.py
      analytics.py       # placement_analytics workflow
```

## Step 1: Create Workflow Engine

**New file**: `backend/services/workflow.py`

```python
from dataclasses import dataclass, field
from typing import Any, Callable, Awaitable
import time, logging

logger = logging.getLogger(__name__)

@dataclass
class StepResult:
    success: bool
    data: Any = None
    error: str | None = None
    elapsed_ms: float = 0

@dataclass
class StepSpec:
    name: str
    step_type: str  # "check", "prepare", "llm_call", "db_write", "respond"
    fn: Callable[..., Awaitable[StepResult]]
    params: dict = field(default_factory=dict)
    optional: bool = False

class Workflow:
    def __init__(self, name: str, steps: list[StepSpec]):
        self.name = name
        self.steps = steps

class WorkflowExecutor:
    def __init__(self, websocket, db, session_id):
        self.ws = websocket
        self.db = db
        self.session_id = session_id
        self.context: dict[str, Any] = {}  # shared state between steps
        self.results: list[dict] = []

    async def execute(self, workflow: Workflow) -> str | None:
        """Run all steps, send progress, return final response text."""
        await self._send_progress(f"Starting {workflow.name}...")

        for step in workflow.steps:
            start = time.monotonic()
            try:
                result = await step.fn(self.context, self.db, **step.params)
                elapsed = (time.monotonic() - start) * 1000
                self.results.append({"step": step.name, "success": result.success, "elapsed_ms": elapsed})

                if not result.success and not step.optional:
                    await self._send_error(f"Failed at step: {step.name}. {result.error}")
                    return None

                if result.data is not None:
                    self.context[step.name] = result.data

            except Exception as e:
                elapsed = (time.monotonic() - start) * 1000
                logger.exception("Workflow step '%s' failed", step.name)
                self.results.append({"step": step.name, "success": False, "elapsed_ms": elapsed})
                await self._send_error(f"Error at step: {step.name}")
                return None

        return self.context.get("response_text")

    async def _send_progress(self, message: str):
        await self.ws.send_json({"type": "assistant_text", "content": message})

    async def _send_error(self, message: str):
        await self.ws.send_json({"type": "error", "content": message})
```

## Step 2: Create Workflow Definitions

Each intent gets a workflow file that defines its steps. Steps are async functions that read/write from the shared `context` dict.

**Example**: `workflows/analyze_job.py`

```python
from services.workflow import StepSpec, Workflow

async def check_profile(ctx, db, **kw):
    from services.profile_service import get_profile
    profile = get_profile(db)
    if not profile:
        return StepResult(success=False, error="No profile found")
    return StepResult(success=True, data=profile)

async def prepare_data(ctx, db, **kw):
    from services.profile_service import profile_to_dict
    from routers.chat import extract_job_description
    profile_dict = profile_to_dict(ctx["check_profile"])
    jd = extract_job_description(kw["user_msg"])
    return StepResult(success=True, data={"profile": profile_dict, "jd": jd})

async def llm_analyze(ctx, db, **kw):
    from services.job_analyzer import analyze_job
    data = ctx["prepare_data"]
    result = await analyze_job(data["jd"], data["profile"])
    return StepResult(success=True, data=result)

async def save_application(ctx, db, **kw):
    from models import Application
    result = ctx["llm_analyze"]
    app = Application(
        company=result.get("company", "Unknown"),
        role=result.get("role", "Unknown"),
        job_description=ctx["prepare_data"]["jd"],
        status="applied",
        cover_letter=result.get("cover_letter", ""),
        recruiter_msg=result.get("recruiter_msg", ""),
        match_score=result.get("match_score", 0.0),
        match_analysis=result.get("match_analysis", ""),
    )
    db.add(app)
    db.commit()
    db.refresh(app)
    return StepResult(success=True, data=app)

async def format_response(ctx, db, **kw):
    result = ctx["llm_analyze"]
    app = ctx["save_application"]
    score_pct = int(result.get("match_score", 0) * 100)
    text = (
        f"Done! Analyzed **{result.get('company')} - {result.get('role')}**\n\n"
        f"**Match Score: {score_pct}%**\n\n"
        f"{result.get('match_analysis', '')[:300]}"
    )
    await kw["websocket"].send_json({"type": "assistant_text", "content": text})
    await kw["websocket"].send_json({"type": "action", "action_type": "application_created", "data": {"application_id": app.id}})
    return StepResult(success=True, data=text)

def get_workflow(user_msg, websocket) -> Workflow:
    return Workflow(name="analyze_job", steps=[
        StepSpec(name="check_profile", step_type="check", fn=check_profile),
        StepSpec(name="prepare_data", step_type="prepare", fn=prepare_data, params={"user_msg": user_msg, "websocket": websocket}),
        StepSpec(name="llm_analyze", step_type="llm_call", fn=llm_analyze),
        StepSpec(name="save_application", step_type="db_write", fn=save_application),
        StepSpec(name="format_response", step_type="respond", fn=format_response, params={"websocket": websocket}),
    ])
```

## Step 3: Workflow Registry

**New file**: `backend/services/workflows/__init__.py`

```python
from . import analyze_job, resume_upload, resume_generate, cover_letter
from . import recruiter_msg, interview_prep, show_applications, show_profile, analytics

WORKFLOWS = {
    "upload_resume": resume_upload,
    "generate_resume": resume_generate,
    "generate_cover_letter": cover_letter,
    "generate_recruiter_msg": recruiter_msg,
    "analyze_job": analyze_job,
    "prepare_interview": interview_prep,
    "show_applications": show_applications,
    "show_profile": show_profile,
    "placement_analytics": analytics,
}

def get_workflow(intent: str, user_msg: str, websocket):
    module = WORKFLOWS.get(intent)
    if module:
        return module.get_workflow(user_msg, websocket)
    return None
```

## Step 4: Refactor _handle_message()

**File**: `backend/routers/chat.py`

Replace the monolithic if/elif chain with:

```python
async def _handle_message(websocket, db, session_id, user_msg):
    # Save user message
    db.add(ChatMessage(session_id=session_id, role="user", content=user_msg))
    db.commit()

    # Detect intent
    intent = detect_intent(user_msg)
    if intent == "general_chat":
        classified = await classify_intent_with_llm(user_msg)
        if classified != "general_chat":
            intent = classified

    # Get workflow
    workflow = get_workflow(intent, user_msg, websocket)
    if workflow:
        executor = WorkflowExecutor(websocket, db, session_id)
        response_text = await executor.execute(workflow)
    else:
        # General chat fallback (streaming)
        response_text = await _handle_general_chat(websocket, db, session_id, user_msg)

    # Save assistant response
    db.add(ChatMessage(session_id=session_id, role="assistant", content=response_text or ""))
    db.commit()
    return response_text
```

The `general_chat` path stays as streaming since it doesn't fit the step model.

## Step 5: Define All 9 Workflows

Each workflow follows the same pattern as the analyze_job example above. The steps for each:

**upload_resume**: check (none) → respond (prompt user to upload)
**generate_resume**: check_profile → prepare_data → llm_generate → format_pdf → respond
**generate_cover_letter**: check_profile → check_app → prepare_data → llm_generate → save → respond
**generate_recruiter_msg**: check_profile → check_app → parse_channel → llm_generate → save → respond
**analyze_job**: check_profile → prepare_data → llm_analyze → save_app → respond
**prepare_interview**: check_apps → check_existing → check_profile → llm_prep → save_prep → respond
**show_applications**: query_apps → format_summary → respond
**show_profile**: check_profile → format_summary → respond
**placement_analytics**: query_stats → llm_narrative → format_dashboard → respond

## Step 6: Progress Updates

The executor sends `{"type": "assistant_text", "content": "..."}` before each step. The client sees:
1. "Starting analyze_job..."
2. "Checking profile..."
3. "Analyzing job description..."
4. "Saving results..."
5. Final response

This gives the user real-time feedback on what's happening.
